Xây dựng công cụ hỗ trợ sinh và thực thi ca kiểm thử chức năng cho REST API dựa trên đặc tả OpenAPI
Mô tả đề tài

Đề tài tập trung xây dựng một công cụ hỗ trợ kiểm thử chức năng cho REST API, trong đó đầu vào là file đặc tả OpenAPI của hệ thống, còn đầu ra là các ca kiểm thử chức năng được sinh tự động, kèm khả năng thực thi và tạo báo cáo kết quả. OpenAPI là chuẩn mô tả giao diện cho HTTP API ở dạng máy có thể đọc được, cho phép cả con người lẫn công cụ hiểu các endpoint, tham số, request body, response và ràng buộc dữ liệu của dịch vụ mà không cần đọc toàn bộ mã nguồn.

Công cụ được đề xuất sẽ phân tích đặc tả OpenAPI để trích xuất các thành phần chính của API như phương thức, đường dẫn, tham số đầu vào, kiểu dữ liệu, trường bắt buộc, giá trị hợp lệ và cấu trúc phản hồi. Từ đó, hệ thống tự động sinh các nhóm ca kiểm thử chức năng gồm ca hợp lệ và không hợp lệ, chẳng hạn như thiếu trường bắt buộc, sai kiểu dữ liệu, giá trị vượt biên, sai enum hoặc dữ liệu rỗng. Sau khi sinh test, công cụ sẽ gửi request đến API, kiểm tra mã trạng thái, đối chiếu response với schema mong đợi và tổng hợp kết quả thành báo cáo pass/fail theo từng endpoint. Việc sinh test tự động từ OpenAPI cũng phù hợp với hướng tiếp cận đã được các công cụ chuyên biệt như Schemathesis sử dụng, trong đó schema được dùng để tạo ra nhiều ca kiểm thử, bao gồm cả edge cases mà kiểm thử thủ công dễ bỏ sót.

Về triển khai, đề tài có thể sử dụng Python làm ngôn ngữ chính; phần thực thi test có thể dựa trên pytest vì framework này hỗ trợ tốt việc tham số hóa test, rất phù hợp với các ca kiểm thử được sinh tự động từ đặc tả API. Nếu muốn nâng chất lượng đề tài, bạn có thể tích hợp thêm cơ chế sinh dữ liệu kiểm thử đa dạng theo hướng property-based testing, vì Schemathesis được xây trên nền Hypothesis và cho phép mở rộng theo kiểu này.
